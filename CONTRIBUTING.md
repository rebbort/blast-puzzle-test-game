# Contributing Guidelines

This project enforces the [Conventional Commits](https://www.conventionalcommits.org/) standard. Commit messages must follow the `<type>: <description>` pattern.

## Valid examples

- `feat: add new puzzle mechanic`
- `fix: resolve crash on start`
- `chore: update dependencies`
- `docs: update README`

## Invalid examples

- `update stuff`
- `bug fixes`

Commits that do not follow the convention will be rejected by the commit-msg hook.
